## Javascript
Следующим на расстрел пошел JS. Решил оставить дизайн окончательный на попозже, сперва надо разобраться с кодом.

Главный принцип - все задачи хранятся в tasks, он обновляется из памяти local storage.

### Глава 1. renderTasks
Итак, начал с того что добавил функцию отрисовки карточек (renderTasks). В нее, с помощью цикла начал отрисовку карточки.
Итак, элементы в карточке рисуются по следующему принципу:
1) Создается константа, начинающая с RT (render Task), в ней лежит тег html
2) Присваивается название класса (если есть)
3) Присваивается контент (если есть)
4) Через appendChild этот тег появляется на сайте.

Из интересного:
- **Удаление**. Добавил кнопку. Ей, в dataset, присвоил такой же id'шник как и у карточки (далее понятно зачем). Добавил event listener. Когда кнопка нажимается, то в tasks переписываются все задачи, с другими айди (отличным от id кнопки). Далее сохраняются в память, и перерисовываются снова
- **Чекбокс**. При его изменении запускается функция, которая меняет в памяти свойство is_completed у объекта на то, в котором находится флажок.

### Глава 2. Добавление задач
Далее занялся добавлением новой задачи. Сделал через промпт, так как цель проекта не красивый сайт, а показать знание js. Там все по принципу:
1) Присваиваем константе, начинающейся с ADD (добавить с англ.) значение из промпта
2)  Все подобные константы перемещаем в ADDtask, присваивая их всем полям задачи.
3) Через push, добавляем ADDtask в массив задач, сохраняемся, рендеримся.

Из интересного, сделал счетчик id'шников: если массив задач пустой - присваивается 1, если НЕ пустой, то все айдишники через map собираются в массив, из них ищется самый большой, и ставится на 1 больше.
Честно, не знаю насколько это правильно и энергоемко, но показалось прикольной идеей, решил реализовать.

### Глава 3. Вкладки
Тут сделано по принципу: нажатие на вкладки меняют кодовое слово в переменной. Ее читает функция рендера, и запускает фильтры .

- Сегодня - выполненные и невыполненные задачи с сегодняшней датой
- Входящие - все невыполненные
- Законченные - все выполненные
- Все - понятно из названия
